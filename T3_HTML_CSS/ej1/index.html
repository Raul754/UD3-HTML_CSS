Secciones y lineas generadas en un documento HTML5

La especificación HTML5 trae muchos nuevos elementos a los desarrolladores web, permitiéndoles descubrir la estructura de un documento web con semantica estandarizada. Este documento describe estos elementos cómo usarlos para definir el perfil de cualquier documento.

Problemas resueltos por HTML5

la definicío de la estructura de un documento HTML4 y su algortmo de perfilado es muy tosco y genera numerosos problemas:

  1. HTML5 quita la necesidad de elementos <div> para definir secciones semánticas sin definir valores especificos para los atributos class, introduciendo un nuevo elemento, <seccion>, el elemento de sección HTML

  2. Mezclar varios documentos es difícil: la inclusion de un sub-documento en un documento principal. Esto se resuelve en HTML5 con los elementos de seccionado (<article>, <section>, <nav> y <aside>) son siempre subsecciones de su sección mas ancestra más cercana.

  3. HTML5 introduce el elemento <hgroup> que oculta todos los elementos de cabecera excepto el primero de más alto rango (por ejemplo, <hgroup> <h1>Justine</h1><h2>Les Malheurs de Vertru</h2><hgroup> crea el perfil 1. Justine)
